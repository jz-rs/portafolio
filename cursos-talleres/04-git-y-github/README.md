# Git y GitHub – Control de Versiones

## Descripción

Curso introductorio sobre control de versiones utilizando Git y la plataforma GitHub. Se aprendieron los conceptos fundamentales para gestionar proyectos de software, trabajar con repositorios y mantener un historial de cambios ordenado y seguro.

## Contenidos vistos

### Conceptos básicos

* Qué es el control de versiones.
* Diferencia entre Git y GitHub.
* Ventajas del uso de control de versiones en proyectos de software.
* Repositorio local vs repositorio remoto.

### Git

* Inicialización de un repositorio (`git init`).
* Estados de los archivos:

  * working directory
  * staging area
  * repository
* Comandos principales:

  * `git status`
  * `git add`
  * `git commit`
  * `git log`
* Creación y uso de commits.
* Buenas prácticas al escribir mensajes de commit.

### GitHub

* Creación de repositorios en GitHub.
* Vinculación de repositorios locales con repositorios remotos.
* Uso de:

  * `git remote`
  * `git push`
  * `git pull`
* Clonado de repositorios (`git clone`).
* Visualización del historial de cambios desde GitHub.

### Flujo de trabajo

* Flujo básico de trabajo con Git:

  1. Modificar archivos
  2. Agregar cambios al staging
  3. Confirmar cambios con commit
  4. Subir cambios al repositorio remoto
* Sincronización entre repositorio local y remoto.

### Ramas (branches)

* Concepto de ramas.
* Creación y cambio de ramas.
* Uso de ramas para desarrollar funcionalidades sin afectar la rama principal.
* Integración de cambios (merge).

### Buenas prácticas

* Commits pequeños y descriptivos.
* Uso frecuente de `git status`.
* Mantener el repositorio organizado.
* Importancia del README en los proyectos.

## Actividades realizadas

* Creación y configuración de repositorios locales.
* Publicación de repositorios en GitHub.
* Subida y actualización de archivos.
* Ejercicios prácticos de versionado y sincronización.
