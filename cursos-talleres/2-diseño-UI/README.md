# Diseño UI / UX – Fundamentos

## Descripción

Curso orientado a comprender los fundamentos del **Diseño de Interfaces de Usuario (UI)** y la **Experiencia de Usuario (UX)**, enfocado en cómo diseñar productos digitales claros, usables y coherentes. Se trabajaron principios de diseño, estructura de interfaces, patrones de interacción, procesos de diseño y sistemas de diseño aplicados a productos digitales como sitios web y aplicaciones.

## Conceptos principales

### UI – User Interface

* Interfaz mediante la cual una persona interactúa con un sistema digital.
* Conjunto de elementos visuales y funcionales que permiten al usuario comunicarse con el sistema.
* Importancia de la claridad visual, la organización y la consistencia.

### UX – User Experience

* Experiencia completa del usuario al interactuar con un producto o servicio.
* Incluye percepciones, emociones y facilidad de uso.
* La UI forma parte de la UX, pero la UX abarca todo el proceso de interacción.

### Frontend y diseño

* El diseñador UI define cómo se verá y se sentirá la interfaz.
* El desarrollo frontend implementa el diseño usando tecnologías web.
* El diseño debe guiarse por criterios de usabilidad y experiencia.

## Diseño gráfico aplicado a interfaces

* Aplicación del diseño gráfico al diseño de interfaces digitales.
* Uso de comunicación visual para transmitir información de forma clara.
* Diseño para sitios web y aplicaciones.
* Diferencias entre diseño para web y diseño para apps.

## Principios de diseño visual

* Jerarquía visual.
* Legibilidad.
* Consistencia.
* Contraste.
* Uso del espacio en blanco.
* Escala y proporción.

## Elementos de una interfaz

* Elementos informativos (pasivos).
* Elementos informativos activos.
* Elementos interactivos.
* Importancia de que cada elemento tenga una función clara.

## Usabilidad y heurísticas

* Visibilidad del estado del sistema.
* Lenguaje claro y familiar para el usuario.
* Control y libertad del usuario.
* Consistencia y estándares.
* Prevención de errores.
* Reconocimiento antes que memorización.
* Flexibilidad y eficiencia de uso.
* Diseño estético y minimalista.
* Ayuda y documentación.

## Arquitectura de la información

* Organización del contenido.
* Agrupación lógica de información.
* Reducción de la carga cognitiva.
* División de interfaces complejas en partes más simples.

## Proceso de diseño UI / UX

1. Comprensión del negocio.
2. Definición de usuarios (user persona).
3. Screen flow (flujo de pantallas).
4. Wireframes.
5. Diseño de alta fidelidad.
6. Prototipado.
7. Feedback.
8. Entrega.

## Wireframes

* Representación esquemática de las pantallas.
* Definición de bloques y distribución del contenido.
* Guía para el diseño visual posterior.

## Diseño de alta fidelidad

* Representación visual cercana al producto final.
* Definición de colores, tipografías y estilos.
* Base para el prototipo y el desarrollo.

## Prototipos

* Simulación del funcionamiento del producto.
* Permiten validar el diseño y obtener feedback.
* No siempre son obligatorios, pero son recomendables.

## Sistemas de diseño

* Conjunto de principios, componentes y guías.
* Reutilización de elementos visuales.
* Consistencia en todo el producto.
* Importancia de documentar los diseños.

## Patrones de diseño

* Soluciones reutilizables a problemas frecuentes.
* Patrones generales y patrones locales.
* Uso de estándares conocidos para facilitar el aprendizaje del usuario.

## Herramientas vistas

* Sketch.
* Adobe XD.
* Figma.
* InVision.
* Framer.
* Photoshop.
* Illustrator.
* After Effects.

## Objetivos del diseño UI / UX

* Que el usuario comprenda la funcionalidad de la interfaz.
* Facilitar la navegación.
* Reducir errores y confusión.
* Priorizar las acciones importantes.
* Ofrecer una experiencia clara, funcional y agradable.
